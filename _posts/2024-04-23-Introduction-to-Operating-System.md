## What is an Operating System?
**운영체제**는 사용자와 하드웨어 사이에 동작하는 프로그램이다. 사용자 관점에서 운영체제는 사용자의 프로그램을 실행하고, 이에 따라 발생하는 문제점을 쉽게 해결해준다. 시스템 관점에서 운영체제는 자원과 프로그램을 관리한다.

따라서 운영체제는 컴퓨터를 **<U>편리</U>**하게 사용하고, 하드웨어를 **<U>효율적</U>**인 방법으로 관리한다.

## Computer System Structure
컴퓨터 시스템 구조는 **<U>하드웨어, 운영체제, 프로그램, 사용자</U>**로 크게 네가지로 나눌 수 있다.
![Four Components of a Computer System](/assets/img/240423/Four Components of a Computer System.png)
* **하드웨어**: 기본적인 컴퓨팅 자원을 제공한다.(ex. CPU, Memory, I/O Devices 등..)
* **운영체제**: 하드웨어의 사용을 관리하고 조정한다.
* **프로그램**: 특정 작업을 수행하도록 설계된 소프트웨어 프로그램
* **사용자**: 컴퓨터를 사용하는 이용자

## Operating System Definition
운영체제의 정의는 다음과 같다.
* **자원 할당자**
    * 컴퓨터에서 모든(물리적, 추상적) 자원을 관리함
    * 물리적 자원: CPU, Memory, HDD 등..
    * 추상적 자원: 파일 시스템, 터미널 드라이버, 프로토콜 등..
* **프로그램 관리자**: 에러와 부적절한 컴퓨터의 사용을 막기 위해 프로그램을 관리함
* **커널**: 컴퓨터 운영 체제의 핵심이 되는 컴퓨터 프로그램으로, 시스템의 모든 것을 완전히 제어

## Operating System Design and Implementation
* 정책: 무엇을 할 것인가?
* 매커니즘: 어떻게 할 것인가?
    * 어떤 작업이나 과정을 실행하기 위해 사용되는 구체적인 방법이나 절차

예시는 다음과 같다.
* CPU Timer 구성
    * 정책: 타이머를 얼마나 설정할 것인가?
    * 메커니즘: 다양한 타이머 구성 방법에 대한 세부사항
* 우선순위 관리
    * 정책: 우선순위 레벨 지정
    * 메커니즘: 우선순위 처리를 위한 세부적인 제어 방법

## Computer Startup
**부트 스트랩 프로그램**(부트 스트랩 로더)는 전원을 키거나 재부팅할 때 로드된다.
* 펌웨어라고 알려진 ROM 혹은 EEPROM에 저장되어 있다.
* CPU 레지스터에서 디바이스 컨트롤러까지 모든 컴퓨터 시스템을 초기화한다.
* 동작 방식: Power → BIOS → 부트 섹터 → OS → Processor
    * BIOS: 하드웨어의 I/O를 조절하는 소프트웨어로, POST검사를 한다.
    * POST(Power on Self Test): 부팅할 때 BIOS가 문제 있는지 하드웨어 기기들을 자가 검사

## Computer System Organization
![A typical PC computer system](/assets/img/240423/A typical PC computer system.png)
CPU와 장치 관리자가 버스(컴퓨터 구성 요소 간에 데이터를 전송하는 데 사용되는 공통 연결 경로)를 통해 연결되며 공유 메모리에 접근할 수 있다. CPU와 장치 관리자는 메모리 사이클을 위해 경쟁한다.
* CPU는 메인 메모리와 버퍼 사이 데이터를 옮긴다.
* I/O는 장치에서 제어 버퍼로 데이터를 옮긴다.
* 장치 관리자
    * 각 장치 관리자는 특정 유형의 장치를 관리하고, 일반적인 버퍼를 갖고 있음
    * 장치 관리자는 CPU에게 **<U>인터럽트</U>**를 발생시킴으로써 작업을 완료했다는 것을 알림 

**32비트와 64비트 컴퓨터 시스템 차이**
* CPU가 데이터를 처리할 때 사용하는 레지스터의 크기
* 32비트는 최대 약 4GB(2^32)까지의 메모리(RAM)만 인식할 수 있는 반면 64비트는 16엑사바이트(2^64)까지 인식 가능

## Common Functions of Interrupts
인터럽트는 **인터럽트 벡터**를 통해 **인터럽트 서비스 루틴(ISR)**으로 제어권을 전달한다. 
* 인터럽트 서비스 루틴(ISR): 인터럽트 처리 동작
* 인터럽트 벡터: 인터럽트가 발생했을 때, 그 인터럽트를 처리할 수 있는 서비스 루틴들의 주소를 가지고 있는 공간
* 인터럽트 아키텍처는 중단된 명령어 주소를 갖고 있어야 함
* 다른 인터럽트가 처리되는 동안 들어오는 인터럽트는 비활성화되어 인터럽트가 손실되는 것을 방지함(ex. 인터럽트 우선순위)
* **트랩**(trap): 오류나 사용자 요청에 의해 소프트웨어적으로 만들어진 인터럽트
* 인터럽트는 하드웨어에서 일반적으로 생성

### Interrupt Comparison
인터럽트는 외부 인터럽트, 내부 인터럽트가 있다.

#### External interrupt
비동기, 하드웨어 장치에 의해 발생

#### Internal interrupt
* **Fault**
    * 문제가 있어서 인터럽트 → 핸들러에 의해서 처리 될 수 있다고 판단되고, 다시 현재의 명령어로 돌아감
    * 현재 명령어 → ISR → ISR 전의 명령어로 돌아감
    * ex) page fault, memory space fault
* **Trap**
    * 사용자가 의도적으로 발생 시키는 것
    * 현재 명령어 → ISR → 트랩된 명령어의 다음 명령어로 돌아감
    * ex) system call, exception handling
* **Abort**
    * 복구할 수 없는 예외 → 복구할 수 없어서 종료
    * ISR → 현재 작업을 중단
    * ex) divide by zero

### Interrupt Handling
운영체제는 레지스터와 프로그램 카운터를 저장함으로써 CPU의 상태를 보존한다. 또한, 발생한 인터럽트의 유형을 결정한다.

* **벡터 인터럽트 시스템(Vector Interrupt system)**: 각각의 인터럽트를 미리 만들어 놓고, 인터럽트가 왔을 때 그 인터럽트에 맞는 미리 만들어 놓은 인터럽트를 찾아 처리하는 방식
* **폴링 시스템(Polling system)**: CPU가 주기적으로 각 장치의 상태를 체크하여 어떤 장치가 인터럽트를 요청했는지 확인하는 방식
    * 장치의 상태는 핸드 쉐이킹(handshaking), 바쁜 대기(busy waiting)를 통해 결정됨
    * 장점: 빠른 응답
    * 단점: 자원(CPU) 낭비, 빠른 I/O장치 필요, 짧은 I/O 서비스 루틴 필요

인터럽트 서비스 루틴은 각각의 인터럽트 유형에 대해 행동해야할 지를 결정하는 전용 코드 영역을 이용한다.

## Storage Structure
* 메인 메모리: CPU가 접근할 수 있는 대용량 저장 매체
* 보조 저장장치(Secondary Storage): 메인 메모리의 확장으로, 큰 용량의 비휘발성 저장 공간을 제공
* 자기 디스크(Magnetic Disks)
* 전자 디스크(Electronics disks): 플래시 메모리로 구성되어 있으며, 전기적으로 데이터를 지우고 다시 기록할 수 있는 비휘발성 컴퓨터 기억 장치(RAM에 비해서는 읽기, 쓰기 속도가 매우 느림)

### Storage Hierarchy
저장 장치는 **<U>속도, 비용, 크기, 휘발성</U>** 등의 기준에 따라 계층적으로 구분된다.
**캐쉬(Cache)**는 자주 필요한 데이터나 값의 복사본을 일시적으로 저장, 보관하기 위해 사용하는 곳이다. **캐싱(Cashing)**은 정보를 더 빠른 저장 장치로 복사하는 과정이다. 
![Storage device hierarchy](/assets/img/240423/Storage-device hierarchy.png)

CPU에서 멀어질 수록 접근 속도는 느리나 용량이 커짐

## Operating System Structure
### Multiprogramming
**멀티 프로그래밍(Multiprogramming)**은 메모리에 여러 개의 응용 프로그램을 적재하는 것이다. 단일 사용자는 CPU와 입출력(I/O) 장치를 항상 바쁘게 유지할 수 없기 때문에, 멀티 프로그래밍은 여러 작업(코드와 데이터)을 조직하여 CPU가 항상 실행할 작업을 갖도록 한다.

![multiprocessing architecture](/assets/img/240423/multiprocessing architecture.png)

* 시스템의 전체 작업 중 일부(나머지)는 메모리에 유지
* 작업 스케줄링을 통해 하나의 작업이 선택되어 실행
* 작업이 대기해야 할 경우, 운영 체제는 다른 작업으로 전환

멀티 프로그래밍은 시스템 자원을 최대한 활용하여 CPU 유휴 시간을 최소화하기 때문에 **효율성**을 위해 필요하다.

### Timesharing(multitasking)
**타임쉐어링(멀티태스킹)**은 CPU가 작업을 매우 자주 전환하여 사용자가 작업이 실행되는 동안 각 작업과 상호 작용할 수 있는 멀티프로그래밍의 논리적인 확장이다.
* 각 사용자는 메모리에서 실행 중인 적어도 하나의 프로그램(프로세스)을 가지고 있다.
* 동시에 실행할 준비가 된 여러 작업이 있는 경우 **CPU 스케줄링**이 필요
* 프로세스가 메모리에 맞지 않을 경우, **스와핑**을 통해 프로세스를 메모리 안팎으로 이동시켜 실행
    * 스와핑(swapping): 운영 체제가 메모리 관리를 위해 사용하는 기법이다. 메모리(RAM)에 할당할 수 있는 공간이 부족할 때, 활성 메모리에서 현재 사용하지 않거나 적게 사용하는 프로세스들의 데이터를 하드 드라이브나 다른 저장 매체의 스왑 공간(swap space)으로 임시로 이동시키는 과정이다.

## Process Management
**프로세스(Process)**는 현재 실행중인 프로그램이다. 프로세스는 시스템 내에서 작업 단위로 기능한다. 프로그램은 **정적**존재이고, 프로세스는 **동적**존재이다.

프로세스는 작업을 완수하기위해 자원(CPU, 메모리, 입출력 장치, 파일, 초기화 데이터 등)이 필요하다. 프로세스 종료시, 재사용 가능한 모든 자원 회수를 요구한다.

일반적으로 시스템은 많은 프로세서들이 있고, 몇몇의 운영체제는 하나 이상의 CPU에서 병렬로 실행한다.

### Process Management Activities
운영체제는 프로세스 관리와 관련하여 다음과 같은 역할을 한다.
* 사용자 및 프로세스 생성 및 삭제
* 프로세스 일시 중지 및 재개
* 프로세스 동기화를 위한 메커니즘 제공
* 프로세스 통신을 위한 메커니즘 제공
* 교착 상태 처리를 위한 메커니즘 제공

## Memory Management
처리 전후의 모든 데이터는 메모리 안에 있다. 메모리안에 실행하기 위한 모든 명령어가 있다.

메모리 활동은 다음과 같다.
* 현재 메모리의 어떤 부분이 사용되고 있는지, 누가 사용하고 있는지 추적
* 프로세스와 데이터를 메모리로 옮기거나 메모리에서 제거할지 결정
* 필요에 따라 메모리 공간을 할당하고 해제

## File Management
운영체제는 정보 저장소에 대해 일관된 논리적인 관점을 제공한다.
* 물리적 속성을 논리적 저장 단위인 **파일**로 추상화 한다.
* 각 저장 장치는 디바이스 드라이버에 의해 제어된다.

## File System & Storage Management
### File System Management
* 파일은 일반적으로 디렉토리로 조직된다.
* 대부분의 시스템에서는 누가 무엇에 접근할 수 있는지 결정하기 위해 접근 제어가 이루어짐

운영체제 파일 시스템 관련 활동은 다음과 같다.
* 파일 및 디렉토리 생성 및 삭제
* 파일 및 디렉토리 조작을 위한 기본 명령어 제공
* 보조 저장소에 파일 매핑
* 비휘발성 저장 장치에 파일 백업

### Storage Management
* 여유 공간 관리
* 저장 공간 할당
* 디스크 스케줄링
